# ADL Console (iOS)

Native SwiftUI companion to the web Data Ops Platform console
(`components/Console/*`). Consumes the local `Packages/ConsoleCore` Swift
package (`ConsoleModels`, `ConsoleAPI`, `ConsoleState`) — no other runtime
dependencies.

## Regenerating the Xcode project

The `.xcodeproj` is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from `project.yml` and is committed so CI/other machines can build without
installing XcodeGen. If you change `project.yml`, targets, file groups, or
add/remove Swift files, regenerate it:

```sh
brew install xcodegen   # once, if not already installed
cd ios-console
xcodegen generate
```

Commit the resulting `ADLConsole.xcodeproj` changes alongside your source
changes.

## Building & testing

```sh
cd ios-console

# Build for the simulator
xcodebuild -project ADLConsole.xcodeproj -scheme ADLConsole \
  -destination 'generic/platform=iOS Simulator' build

# Run unit tests on a concrete simulator (pick one from `xcrun simctl list devices available`)
xcodebuild -project ADLConsole.xcodeproj -scheme ADLConsole \
  -destination 'platform=iOS Simulator,id=<SIMULATOR-UDID>' test
```

The `ConsoleCore` package itself is tested independently:

```sh
cd ios-console/Packages/ConsoleCore
swift test
```

## Structure

```
ios-console/
  project.yml                 # XcodeGen project spec (source of truth)
  ADLConsole.xcodeproj/        # generated — do not hand-edit
  ADLConsole/                  # app target sources
    ADLConsoleApp.swift        # @main entry point
    DesignSystem/               # brand color/font tokens + reusable views
    Support/                    # ConsoleLanguage (bilingual t() helper)
    Auth/                       # AuthService protocol/stub + AuthView
    State/                      # AppState (ObservableObject)
    Shell/                      # RootView, ConsoleShellView, nav destinations
    Screens/                    # OverviewView, PlaceholderScreenView
    Assets.xcassets/
  ADLConsoleTests/             # unit tests (view-model/logic layer only)
  Packages/ConsoleCore/        # local SPM package (owned by an earlier task)
```

## Auth status

Sign-in currently goes through `StubAuthService` (see the TODO in
`ADLConsole/Auth/AuthService.swift`) — it validates input shape only and
always succeeds. The real cookie-session handshake against
`/api/auth/csrf` → `/api/auth/callback/credentials` → `/api/auth/session`
(mirroring `lib/client/auth.ts`) is intentionally deferred to a later task
that can validate against a live preview deployment.
