# Comprehensive UI/UX Audit Report — African Data Layer

**Date**: 2026-03-14
**Audited by**: Charles
**Scope**: All 28 TSX components, global styles, HTML entry point, Tailwind config

---

## Anti-Patterns Verdict

**PASS with minor concerns.** This does NOT immediately read as AI-generated. The design has a clear, intentional identity — a civic-tech mobile app with a cohesive navy/gold/terracotta palette and consistent design language. However, several AI slop signals are present:

- **Gradient progress bars** (`bg-gradient-to-r from-[#0f2b46] to-[#4c7c59]`) used in 3+ places — decorative gradient on a utilitarian element
- **Identical card grids** — DailyProgressWidget uses a 2x2 grid of same-sized stat cards (icon + label + big number), a known AI template pattern
- **rounded-2xl cards everywhere** — nearly every container is `rounded-2xl border border-gray-100 shadow-sm`, creating visual monotony
- **Inter font** — explicitly called out in the DON'T guidelines as overused
- **`text-[10px] font-bold uppercase tracking-widest`** pattern repeated ~50+ times — while intentional, it's become a signature of AI-generated interfaces

The app avoids the worst offenders: no glassmorphism abuse, no cyan-on-dark, no gradient text, no bounce easing. The palette is distinctive and appropriate for the African market context.

### Top 5 Critical Issues
1. **`user-scalable=no` blocks zoom** — WCAG 1.4.4 violation, prevents low-vision users from zooming
2. **Missing ARIA labels on many interactive elements** — buttons, toggles, and navigation items lack accessible names
3. **Multiple buttons without discernible text** — back buttons, settings items with no `aria-label`
4. **Hard-coded color values throughout** — no design tokens, 30+ unique hex values scattered across components
5. **Touch targets too small** — several 8px/10px icons used as or inside interactive elements

### Overall Quality Score: **6.5/10**

The app is functional and has a clear visual identity, but accessibility gaps and lack of design tokens create maintainability and inclusivity risks.

## Detailed Findings by Severity

### Critical Issues

**C1. `user-scalable=no` prevents zoom**
- **Location**: `index.html:6`
- **Category**: Accessibility
- **Description**: `maximum-scale=1.0, user-scalable=no` prevents users from pinch-to-zoom
- **Impact**: Blocks low-vision users from enlarging content. Violates WCAG 1.4.4 (Resize Text, Level AA)
- **WCAG**: 1.4.4 Resize Text (AA)
- **Recommendation**: Remove `maximum-scale=1.0` and `user-scalable=no`
- **Suggested command**: `/harden`

**C2. LevelUpCelebration click-to-dismiss on overlay has no keyboard equivalent**
- **Location**: `components/LevelUpCelebration.tsx:58`
- **Category**: Accessibility
- **Description**: The celebration overlay uses `onClick={onDismiss}` on a `<div>` — not focusable, no keyboard handler, no role
- **Impact**: Keyboard-only users are trapped in the overlay for 3 seconds with no way to dismiss
- **WCAG**: 2.1.1 Keyboard (A)
- **Recommendation**: Add `role="dialog"`, `aria-label`, `onKeyDown` for Escape, and make it focusable
- **Suggested command**: `/harden`

**C3. Settings toggle has no accessible state announcement**
- **Location**: `components/Screens/Settings.tsx:133-139`
- **Category**: Accessibility
- **Description**: The high-contrast toggle is a `<button>` with a visual circle, but no `role="switch"`, no `aria-checked`, no label
- **Impact**: Screen reader users cannot determine the toggle's purpose or state
- **WCAG**: 4.1.2 Name, Role, Value (A)
- **Recommendation**: Add `role="switch"`, `aria-checked={highContrast}`, and `aria-label`
- **Suggested command**: `/harden`

### High-Severity Issues

**H1. Back buttons lack aria-labels across all screens**
- **Location**: `Details.tsx:54`, `Settings.tsx:41`, `RewardsCatalog.tsx:45`, `Analytics.tsx` (similar pattern)
- **Category**: Accessibility
- **Description**: Back buttons contain only an `<ArrowLeft>` icon with no accessible text
- **Impact**: Screen readers announce "button" with no context
- **WCAG**: 1.1.1 Non-text Content (A)
- **Recommendation**: Add `aria-label={t('Go back', 'Retour')}`
- **Suggested command**: `/harden`

**H2. Navigation buttons have no `aria-current` for active state**
- **Location**: `components/Navigation.tsx:45-55`
- **Category**: Accessibility
- **Description**: Active tab is indicated only by color change (`text-[#0f2b46]` vs `text-gray-400`). No `aria-current="page"` or role
- **Impact**: Screen reader users can't determine which tab is active. Color-only indication also fails WCAG 1.4.1
- **WCAG**: 1.3.1 Info and Relationships (A)
- **Recommendation**: Add `aria-current={isActive ? 'page' : undefined}` and `role="tablist"`/`role="tab"`
- **Suggested command**: `/harden`

**H3. Inter font — overused, violates typography guidelines**
- **Location**: `index.html:11`, `index.css:32`
- **Category**: Anti-Pattern
- **Description**: Inter is explicitly listed in the DON'T guidelines as overused
- **Impact**: Makes the app look generic
- **Recommendation**: Consider a distinctive alternative (e.g., DM Sans, Plus Jakarta Sans, or a pan-African typeface)
- **Suggested command**: `/normalize`

**H4. 30+ hard-coded hex colors with no design tokens**
- **Location**: Throughout all components
- **Category**: Theming
- **Description**: Colors like `#0f2b46`, `#c86b4a`, `#4c7c59`, `#f9fafb`, `#1f2933`, `#f2f6fa`, `#e7eef4`, `#d5e1eb`, `#fff8f4`, `#f7f4ff`, `#6b46c1`, `#c53030` etc. are hard-coded inline
- **Impact**: No single source of truth. Theme changes require find-and-replace across 28 files. Dark mode impossible without tokens
- **Recommendation**: Extract to CSS custom properties or Tailwind theme config
- **Suggested command**: `/extract` then `/normalize`

**H5. `<img>` fallback uses external URL without lazy loading**
- **Location**: `components/Screens/Details.tsx:64`
- **Category**: Performance
- **Description**: `src={point.photoUrl || \`https://picsum.photos/seed/${point.id}/800/400?grayscale&blur=2\`}` loads an external image as fallback with no `loading="lazy"`
- **Impact**: Unnecessary network request for placeholder images, especially on low-bandwidth African networks
- **Recommendation**: Use a local placeholder SVG/CSS gradient instead; add `loading="lazy"` to all images
- **Suggested command**: `/optimize`

**H6. Suspense fallback is a styled card — should be a skeleton**
- **Location**: `App.tsx:413-416`, `Home.tsx:626-630`
- **Category**: Performance / UX
- **Description**: Fallback is a static "Loading screen..." text card instead of a skeleton loader
- **Impact**: Jarring transition when content loads — users see a small card then a full screen
- **Recommendation**: Use content-shaped skeleton placeholders
- **Suggested command**: `/optimize`

**H7. Navigation `<nav>` has no `aria-label`**
- **Location**: `components/Navigation.tsx:40`
- **Category**: Accessibility
- **Description**: The `<nav>` element has no accessible name to distinguish it from other navigation landmarks
- **WCAG**: 1.3.1 Info and Relationships (A)
- **Recommendation**: Add `aria-label={t('Main navigation', 'Navigation principale')}`
- **Suggested command**: `/harden`

**H8. Settings buttons are non-functional (no onClick handlers)**
- **Location**: `components/Screens/Settings.tsx:52-66`, lines 59, 72, 82, 95, 147, 154
- **Category**: UX
- **Description**: "Edit Profile Info", "Security & Password", "Offline Sync Settings", "Data Usage & Limits", "Notifications", "Privacy Terms", "Data Usage" — all are `<button>` elements with `hover` states but NO `onClick` handlers
- **Impact**: Users tap expecting action and nothing happens. Misleading interactive affordance
- **Recommendation**: Either wire up the handlers or remove the interactive styling and add "Coming soon" badges
- **Suggested command**: `/clarify`

### Medium-Severity Issues

**M1. Repeated `text-[10px]` creates accessibility risk**
- **Location**: Throughout all components (50+ instances)
- **Category**: Accessibility
- **Description**: 10px text (below 12px minimum recommended) used extensively for labels, metadata, badges
- **Impact**: Difficult to read on mobile, especially in bright African sunlight. Below WCAG recommendation
- **Recommendation**: Increase minimum text to 11px or 12px
- **Suggested command**: `/adapt`

**M2. Gradient progress bars are decorative, not informational**
- **Location**: `DailyProgressWidget.tsx:78`, `XPPopup.tsx:78`, `Home.tsx:587`
- **Category**: Anti-Pattern
- **Description**: `bg-gradient-to-r from-[#0f2b46] to-[#4c7c59]` on progress bars adds no information
- **Impact**: Minor AI slop signal; gradient makes it harder to perceive progress at-a-glance
- **Recommendation**: Use a single solid color for progress bars
- **Suggested command**: `/distill`

**M3. No dark mode support**
- **Location**: Global
- **Category**: Theming
- **Description**: Background is hard-coded `#f9fafb`, no `@media (prefers-color-scheme: dark)` support
- **Impact**: Poor usability at night; no respect for OS-level dark mode preference
- **Recommendation**: Implement dark mode using CSS custom properties
- **Suggested command**: `/normalize`

**M4. `fixed` positioned buttons may overlap content on small screens**
- **Location**: `Home.tsx:695-736`
- **Category**: Responsive
- **Description**: The view toggle and contribute FAB use `fixed` positioning with `bottom-[calc(6rem+var(--safe-bottom))]` — on very short screens (e.g., landscape mobile), these may overlap the list content
- **Impact**: Content obscured on small viewports
- **Recommendation**: Use `sticky` or add bottom padding to scrollable content
- **Suggested command**: `/adapt`

**M5. Vertical picker dropdown has no outside-click dismiss**
- **Location**: `Home.tsx:536-562`
- **Category**: UX
- **Description**: The category dropdown (`isVerticalPickerOpen`) only closes when a category is selected or the toggle is clicked again
- **Impact**: Users expect clicking outside to close dropdowns
- **Recommendation**: Add click-outside handler
- **Suggested command**: `/harden`

**M6. `localStorage` accessed without try/catch**
- **Location**: `App.tsx:67`, `Settings.tsx:26`
- **Category**: Performance / Resilience
- **Description**: `localStorage.getItem()` can throw in private browsing mode on some browsers
- **Impact**: App could crash on first load in private browsing
- **Recommendation**: Wrap in try/catch
- **Suggested command**: `/harden`

**M7. SyncStatusBar uses string interpolation for failed count**
- **Location**: `components/SyncStatusBar.tsx:35`
- **Category**: UX Writing
- **Description**: `${failed} failed — Tap to view` — inconsistent with the "OFFLINE" uppercase in the next state
- **Impact**: Minor inconsistency in status messaging
- **Recommendation**: Standardize case and tone across all states
- **Suggested command**: `/clarify`

**M8. Day labels in StreakTracker are English-only**
- **Location**: `components/StreakTracker.tsx:10`
- **Category**: i18n
- **Description**: `const DAY_LABELS = ['M', 'T', 'W', 'T', 'F', 'S', 'S']` — these are always English regardless of language setting
- **Impact**: French users see English day abbreviations
- **Recommendation**: Localize: `['L', 'M', 'M', 'J', 'V', 'S', 'D']` for French
- **Suggested command**: `/harden`

**M9. Confetti pieces are not reduced-motion aware**
- **Location**: `components/LevelUpCelebration.tsx:30-53`, `index.css:135-144`
- **Category**: Accessibility
- **Description**: No `@media (prefers-reduced-motion: reduce)` check for confetti or other animations
- **Impact**: Users with vestibular disorders may experience discomfort
- **WCAG**: 2.3.3 Animation from Interactions (AAA)
- **Recommendation**: Add `prefers-reduced-motion` media query
- **Suggested command**: `/harden`

**M10. Every card uses identical styling**
- **Location**: Throughout all components
- **Category**: Anti-Pattern
- **Description**: `rounded-2xl border border-gray-100 shadow-sm` appears in virtually every container
- **Impact**: Visual monotony — everything looks the same, reducing information hierarchy
- **Recommendation**: Vary container treatments: some cards, some dividers, some borderless
- **Suggested command**: `/distill`

**M11. `backdrop-blur` used without fallback**
- **Location**: `Home.tsx:740` (`bg-white/95 backdrop-blur`)
- **Category**: Performance
- **Description**: `backdrop-blur` is expensive on mobile and not supported on older Android WebViews
- **Impact**: Performance hit on low-end devices; no visual fallback
- **Recommendation**: Add `@supports` fallback or use solid background
- **Suggested command**: `/optimize`

**M12. Missing `<h1>` on most screens**
- **Location**: Most screen components (Details, Settings, Analytics, etc.)
- **Category**: Accessibility
- **Description**: Screens use `<h3>` or `<h4>` for their titles without a proper heading hierarchy starting at `<h1>`
- **Impact**: Screen readers cannot navigate by heading structure
- **WCAG**: 1.3.1 Info and Relationships (A)
- **Recommendation**: Use proper heading hierarchy starting with `<h1>` per screen
- **Suggested command**: `/harden`

## Patterns & Systemic Issues

1. **No design tokens**: 30+ hard-coded hex values scattered across 28 components. The palette (`#0f2b46`, `#c86b4a`, `#4c7c59`, `#f4c317`) is consistent but not systematized
2. **Missing ARIA throughout**: Back buttons, toggles, navigation tabs, and overlays consistently lack accessible names and roles
3. **`text-[10px]` epidemic**: Used 50+ times as the default "small label" size — below readable minimums
4. **Identical card treatment**: `rounded-2xl border border-gray-100 shadow-sm` is the only container pattern — no hierarchy variation
5. **French localization incomplete**: Day labels, accented characters, and some copy lack proper French translation
6. **No `prefers-reduced-motion` support**: All animations play regardless of user preference

## Positive Findings

1. **Strong offline-first architecture**: IndexedDB queue with sync status bar is well-implemented and genuinely useful for the target market
2. **Good lazy loading of screens**: `React.lazy()` for all non-critical screens reduces initial bundle
3. **Cohesive color palette**: Navy/terracotta/green is distinctive, appropriate for the African market, and consistently applied
4. **Safe area inset handling**: Proper `env(safe-area-inset-*)` support for notched devices
5. **Low-end device detection**: Automatic "Lite Mode" for low-end devices is excellent for the target demographic
6. **Bilingual support**: EN/FR toggle is well-integrated throughout (despite accent issues)
7. **Security awareness**: Fraud detection, GPS integrity checks, and EXIF validation show mature security thinking
8. **High contrast mode**: Available toggle in settings — good accessibility starting point
9. **ErrorBoundary wrapping**: App-level error boundary prevents white-screen crashes
10. **`aria-hidden="true"` on decorative SVG**: BrandLogo correctly marks the SVG as decorative

## Recommendations by Priority

### 1. Immediate (this sprint)
- Remove `user-scalable=no` from viewport meta tag
- Add `aria-label` to all icon-only buttons (back, contribute, profile, refresh)
- Add `role="switch"` and `aria-checked` to the high-contrast toggle
- Fix LevelUpCelebration keyboard trap (add Escape handler, dialog role)
- Add `aria-current="page"` to active navigation tab

### 2. Short-term (next sprint)
- Extract all hard-coded colors to CSS custom properties or Tailwind theme
- Increase minimum text size from 10px to 12px
- Add `@media (prefers-reduced-motion: reduce)` to disable animations
- Fix French translations (add accents, localize day labels)
- Add click-outside handler for vertical picker dropdown
- Replace `picsum.photos` fallback with local placeholder
- Wrap `localStorage` calls in try/catch

### 3. Medium-term (next 2-4 weeks)
- Replace Inter with a more distinctive typeface
- Implement dark mode via CSS custom properties
- Add skeleton loaders for Suspense fallbacks
- Vary container treatments to reduce card monotony
- Fix proper heading hierarchy (`<h1>` per screen)
- Wire up or disable non-functional Settings buttons

### 4. Long-term (backlog)
- Add `@supports` fallback for `backdrop-blur`
- Pull version string from build config
- Add scroll affordance indicators where scrollbars are hidden
- Implement dynamic "last synced" timestamp in Settings
